# Рабочий процесс: вёрстка новых страниц из Figma

## Цель
Свести правки к минимуму: делаем страницы пиксель-в-пиксель по макету, проверяем до сдачи.

## Что нужно от заказчика перед стартом

1. **URL страницы**
   - Точный slug (`/stomatologiya/`, `/laboratoriya/` и т.д.).
   - Будет ли страница главной? Если да — сразу предупреждаем, чтобы не путать URL.

2. **Figma**
   - Ссылка на файл.
   - Node ID desktop-версии страницы.
   - Node ID мобильной версии страницы (если есть отдельный фрейм).
   - Список секций по порядку (например: Hero → Кому подходит → Что входит → Почему → Задачи → Партнёры → Проекты → CTA).

3. **Код из Figma**
   - Для каждой секции отдельным блоком (как ты прислал для Hero и Supplies). Лучше не весь макет одним списком, а по секциям — так проще не упустить детали.

4. **Ассеты**
   - Все картинки и иконки в SVG/PNG отдельными файлами.
   - Если иконки — из библиотеки (mingcute, mdi, ion, tabler, weui и т.д.), можно дать названия, я возьму SVG сам.
   - Логотипы партнёров в PNG/SVG.

5. **Контент**
   - Тексты, телефоны, e-mail, адрес, если отличаются от текущих.

## Этапы работы

### Этап 1 — Прототип в статичном HTML
- Для каждой секции создаю отдельный файл-прототип (`/ prototypes/lab-hero.html`, `/prototypes/lab-tasks.html` и т.д.).
- Позиционирую элементы строго по коду из Figma.
- Скриншот десктоп + мобилка.
- Сравниваю с макетом и правлю **до** переноса в WordPress.

### Этап 2 — Интеграция в WordPress
- Создаю шаблон страницы (`page-lab.php`, `page-stomatology.php` и т.д.).
- Добавляю ACF-поля под редактируемый контент (заголовки, тексты, картинки, списки).
- Подключаю стили в `main.css` с комментариями по секциям.
- Поднимаю `TRIMED_VERSION`.

### Этап 3 — Проверка
- Скриншоты desktop 1440px, mobile 375px.
- Проверяю URL, редиректы, формы, кнопки, кэш.
- Прохожу чек-лист ниже.

### Этап 4 — Сдача
- Показываю скриншоты секций.
- Если всё ок — коммичу и перехожу к следующей секции/странице.

## Чек-лист перед сдачей

- [ ] Все секции совпадают с Figma на desktop.
- [ ] Все секции совпадают с Figma на mobile.
- [ ] Страница открывается по нужному URL без 301/404.
- [ ] Меню и подвал корректны.
- [ ] Формы и кнопки работают (якоря, submit).
- [ ] Нет горизонтального скролла на mobile.
- [ ] `TRIMED_VERSION` поднят, кэш сброшен.
- [ ] Адаптив не сломал существующие страницы.

## Как общаемся

- **По секциям, а не всё сразу.** Сначала Hero, потом следующая секция.
- **Если есть косяк** — пиши название секции и что именно не так (можно скриншотом с пометками).
- **Если всё ок** — короткое «Ок, вперёд».
