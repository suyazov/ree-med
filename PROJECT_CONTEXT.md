## Разделы лендинга

1. **Hero** — заголовок, описание, преимущества, CTA, изображение, бейджи.
2. **Кому подходит** — сетка карточек с аудиториями, каждая карточка имеет стрелку ↗ в левом нижнем углу.
3. **Что мы поставляем** — круговая диаграмма с позициями (на мобильных превращается в список).
4. **Что входит** — карточки этапов работы.
5. **Задачи клиентов** — зелёный блок + список задач.
6. **Почему выбирают ТриМед** — преимущества, статистика, склад.
7. **Реализованные проекты** — карточка проекта.
8. **Партнёры** — карточки производителей.
9. **FAQ** — аккордеон вопросов/ответов.
10. **Форма заявки** — форма обратной связи с AJAX-отправкой.
11. **Footer** — карта (статичное изображение) + карточка с контактами.

## Редактирование контента

Всё содержимое страницы редактируется через админку WordPress:

1. Перейти в **Страницы → Дезинфекция**.
2. Внизу классического редактора отобразятся поля ACF, сгруппированные по вкладкам (Hero, Кому подходит, Что мы поставляем, Что входит, Задачи клиентов, Почему выбирают, Проекты, Партнёры, FAQ, Форма заявки).
3. В каждой вкладке доступны заголовки, описания и repeater-поля соответствующей секции.
4. Внести изменения и нажать **Обновить**.

### ACF Repeater-поля

- `hero_badges` — бейджи в hero.
- `hero_features` — преимущества hero (иконка + текст).
- `audience_cards` — карточки аудиторий (название, изображение, стиль).
- `supplies_items` — позиции поставок.
- `included_cards` — карточки "Что входит" (изображение, номер, заголовок, флаг зелёной карточки).
- `tasks_list` — список задач (иконка + текст).
- `why_features` — преимущества в секции "Почему выбирают".
- `partners` — партнёры (логотип, название, описание).
- `faq_items` — вопросы/ответы (флаг "открыт по умолчанию").

Если repeater-поле не заполнено, на сайте выводится fallback-контент по умолчанию.

## Ресурсы

- **Источник дизайна:** Figma-файл "ТРИМЕД-for-AS"
- **Десктопный фрейм:** `143:12` (1440px)
- **Мобильный фрейм:** `164:557` (375px)
- **Изображения:** скачаны через Figma API, являются placeholder'ами, подлежат замене на финальные.

### Figma API

- **Токен:** хранится в файле `.env` (не коммитится в репозиторий).
- **Ключ файла:** заполните `FIGMA_FILE_KEY` в `.env` (берётся из URL Figma-файла после `figma.com/design/`).
- **Node ID для Hero (desktop):** `143:12`.
- **Node ID для мобильной версии:** `164:557`.

Пример запроса к Figma API:
```bash
curl -H "X-Figma-Token: $FIGMA_API_KEY" \
  "https://api.figma.com/v1/files/$FIGMA_FILE_KEY/nodes?ids=143:12"
```

## Важные замечания

- Все фото и тексты — мокапы. Клиенту нужно будет заменить их на реальные.
- Карта в футере — статичное изображение `footer-map.png`. При необходимости заменить на интерактивную (Яндекс/Google Maps).
- Иконки соцсетей в футере — placeholder'ы до подтверждения мессенджеров.
- Тема использует кэширование версий CSS/JS через константу `TRIMED_VERSION` в `functions.php`. При каждом изменении стилей/скриптов рекомендуется увеличивать версию.
